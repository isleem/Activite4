Chapitre 02:
Les Réseaux informatiques
1. Réseau informatique (Network) : Un réseau informatique est un ensemble d'ordinateurs et de périphériques connectés entre eux, situés dans une zone géographique donnée, et utilisant des règles spécifiques pour échanger des informations numériques. La mise en réseau est le processus qui permet de relier ces ordinateurs afin qu'ils puissent partager des ressources. Les réseaux informatiques ont évolué pour répondre au besoin de communication entre terminaux distants et ordinateurs. Initialement utilisés pour le transport de données, les réseaux actuels intègrent également la voix et la vidéo. Deux notions fondamentales soutiennent les réseaux : l'interconnexion, qui assure la transmission des données entre les nœuds, et la communication, qui facilite l'échange de données entre les processus en cours d'exécution. Les nœuds représentent les extrémités des connexions, tandis que les processus sont les programmes actifs impliqués dans les communications au sein du réseau informatique. 2. Les caractéristiques d’un réseau : Un réseau informatique comprend plusieurs aspects importants : la topologie (physique et logique), le débit (mesure de la quantité de données transmises par seconde), la distance maximale (portée) et le nombre maximum de nœuds interconnectés. 3. Quelques terminologies en réseau : • Station de travail : machine capable d'envoyer des données vers les réseaux. • Nœud : station de travail, imprimante, serveur ou entité adressable par un numéro unique. • Serveur : entité centrale fournissant des services spécifiques. • Paquet : plus petite unité d'information transmise sur le réseau, contenant l'adresse de l'émetteur, du destinataire et les données. • Réseaux homogènes : ordinateurs du même fabricant (ex : AppleTalk). • Réseaux hétérogènes : ordinateurs de fabricants divers (ex : Ethernet). • Débit : quantité de données transmises par seconde (bit/s ou bps). 4. Éléments d'un réseau : Les éléments d'un réseau informatique incluent les ordinateurs avec une carte de communication, les logiciels (navigateurs, clients de messagerie, serveurs web), les supports de communication (câbles, prises RJ45, Wi-Fi, ligne téléphonique, ADSL, fibre optique), et les équipements d'interconnexion (répéteurs, concentrateurs, commutateurs, routeurs). 5. Classification des réseaux : Les réseaux informatiques peuvent être classifiés selon différents critères. En termes de portée, on distingue les LAN (Local Area Network), limités à une entreprise avec une centaine de machines et des vitesses de transmission allant de 10 à 100 Mb/s. Les MAN (Métropolitain Area Network) regroupent plusieurs LAN dans un même périmètre, utilisant généralement du câble coaxial. Les WAN (Wide Area Network) sont des réseaux étendus, publics, qui assurent la transmission de données sur de longues distances à l'échelle nationale ou mondiale En fonction de l'utilisation, on distingue les Intranets (réseaux internes privés), les Extranets (réseaux internes et externes ouverts vers l'extérieur) et Internet (réseau public mondial interconnectant des réseaux). Les réseaux peuvent être filaires (câbles coaxiaux, paires torsadées, fibre optique, lignes téléphoniques) ou sans fil (ondes radio, infrarouges, satellites, Bluetooth). En termes d'organisation, on trouve les réseaux Peer-to-Peer (chaque ordinateur est à la fois serveur et client) et les réseaux Client/serveur (la plupart des stations sont des clients et certaines sont dédiées à des tâches spécialisées). 6. Types des réseaux : Les réseaux informatiques peuvent adopter différentes topologies, telles que le bus, l'anneau, l'étoile et la maillée. Dans les réseaux en bus, les machines sont reliées en série par un câble coaxial, avec des limitations de débit. Les réseaux en étoile utilisent un commutateur central pour connecter toutes les stations, permettant une transmission sélective des données. Les réseaux en anneau fonctionnent en boucle, où les ordinateurs communiquent les uns après les autres en utilisant un jeton. Les réseaux maillés, souvent utilisés dans les réseaux WAN, garantissent la stabilité en utilisant plusieurs chemins de transfert entre les nœuds. Internet adopte une topologie maillée qui combine des éléments d'une architecture en étoile et d'une connexion en boucle. Ces différentes topologies offrent des options variées pour la configuration et la gestion des réseaux informatiques.
Chapitre 03:
Le modèle OSI
Les cartes réseau permettent aux ordinateurs de se connecter à un réseau et de transmettre/recevoir des données. Les switches permettent la communication entre plusieurs appareils en utilisant des adresses MAC et sont plus efficaces que les hubs, qui transmettent toutes les données reçues à tous les appareils connectés. Les switches sont utilisés pour la gestion du trafic réseau et permettent une communication plus rapide et sécurisée entre les appareils connectés. Modèle OSI •Définit un modèle en sept couches réseau sont les suivantes : La couche Physique La couche Liaison de données La couche Réseau La couche Transport La couche Session La couche Présentation La couche Application •Chaque couche N utilise la couche N-1 et fournit des services à la couche N+1 . •Est très peu implémenté . •Il sert toujours de référence pour identifier le niveau de fonctionnement d'un composant réseau. Modèle TCP/IP • Définit un modèle en quatre couches réseau sont les suivantes : La couche Accès réseau La couche lnternet La couche Transport La couche Application •Chaque couche dispose de fonctionnalités qui lui sont propres fournit des services aux couches immédiatement adjacentes . •Est un modèle de référence pour les communications de réseau moderne. • Represente d'une certaine façon l'ensemble des règles de communication sur lnternet et se fonde sur la notion d'adressage IP. L'adressage est un ensemble de règles qui permettent d'identifier les deux unités en communication dans un réseau. Dans ce contexte, une adresse IP (Internet Protocol) est un numéro qui identifie chaque ordinateur dans le réseau. Il y a deux types d'adressage : IPV4 et IPV6. Dans IPV4, chaque paquet contient une adresse source et une adresse destinataire sur 32 bits. Dans IPV6, les adresses sont codées sur 128 bits et sont représentées textuellement en découpant le mot de 128 bits en 8 mots de 16 bits séparés par le caractère « : », chacun d'eux étant en hexadécimal. Il existe deux types d'adressage IP : IPv4 et IPv6. IPv4 est composé de 4 octets, divisés en deux parties : la partie réseau et la partie hôte. Un réseau est identifié par tous les bits de valeur supérieure de l'adresse IP égaux à la même valeur. Le nombre de bits contenus dans la partie hôte détermine le nombre de machines possibles sur le réseau. Il existe trois types d'adresses IPv4 : l'adresse réseau (tous les bits de la partie hôte sont à zéro), l'adresse de diffusion (tous les bits de la partie hôte sont à un) et l'adresse d'hôte (les valeurs sont situées entre l'adresse réseau et l'adresse de diffusion). Le masque de réseau et de sous-réseau est utilisé pour identifier le réseau et les sous-réseaux.

Chapitre 05:
Système de Gestion de version
1/Qu’est-ce que Git ? Git et GitHub sont des outils de gestion de version populaires et essentiels pour les développeurs. 2/Qu’est-ce que GitHub ? Git est un logiciel de gestion de version qui enregistre les modifications et permet de revenir à des versions antérieures. GitHub est un service en ligne qui héberge les dépôts Git. Git est utilisé localement, tandis que GitHub sert de serveur central pour les dépôts. 3/Utiliser Git : ligne de commande, console et interface graphique La ligne de commande est privilégiée dans ce cours pour Git pour son accès complet aux commandes, sa cohérence entre les participants et sa compatibilité avec d'autres interfaces graphiques. Pour ouvrir la ligne de commande dans Windows, utilisez Windows + R, tapez "cmd" dans la boîte de dialogue "Exécuter" et appuyez sur Entrée. 4/Démarrer un dépôt Git Git vous permet de créer des dépôts pour gérer les fichiers de votre projet. Vous pouvez importer un répertoire existant dans Git ou cloner un dépôt Git déjà existant. Comprendre le fonctionnement de Git et sa gestion des informations est essentiel pour utiliser efficacement cet outil. 5/La gestion des informations selon Git Git enregistre des instantanés des fichiers à chaque étape, stocke ces instantanés localement et permet des opérations rapides sans nécessiter une connexion à un serveur distant. 6/Les zones de travail Git a trois zones de travail : le répertoire de travail, la zone d'index et le répertoire Git. Le répertoire de travail contient les fichiers du projet pour être modifiés. La zone d'index stocke les informations sur les fichiers pour le prochain commit. Le répertoire Git stocke les méta-données et la base de données des objets. Le processus implique de modifier les fichiers dans le répertoire de travail, les indexer dans la zone d'index et valider les modifications pour les ajouter à Git. 7/Créer un dépôt Git à partir d’un répertoire existant Lorsque vous démarrez avec Git, vous pouvez importer un projet existant en utilisant la commande "git init". Cela crée un sous-répertoire ".git" qui contient les fichiers nécessaires au fonctionnement de Git. Ensuite, vous pouvez utiliser la commande "git status" pour vérifier l'état des fichiers du répertoire. Pour indexer des fichiers, utilisez la commande "git add" suivie du nom du fichier ou du répertoire. Vous pouvez également utiliser des fileglobs pour ajouter plusieurs fichiers correspondant à un schéma. Une fois les fichiers indexés, utilisez la commande "git commit" pour les valider et les ajouter en base. Lors du commit, il est recommandé de fournir un message décrivant les modifications effectuées. Enfin, vous pouvez utiliser à nouveau "git status" pour vérifier que tous les fichiers sont sous suivi de version et qu'aucune modification n'a été apportée depuis le dernier commit. 8/Consulter l’historique des modifications Git La commande "git log" affiche l'historique des commits avec les informations clés. C'est utile pour consulter les modifications enregistrées dans le dépôt Git. 9/Annuler des modifications apportées à un fichier Roll back allows you to revert to a previous state of a project recorded in a version control system. In Git, you can use the "git checkout -- filename" or "git restore" commands to return to the previous state of a file after making changes. 10/Qu’est ce qu’une branche ? Les branches Git permettent de travailler sur de nouvelles fonctionnalités sans affecter le projet principal. Une branche est un pointeur léger vers un commit. Git facilite la création et la fusion des branches. Le commit contient des informations sur l'auteur, le message et les commits précédents. La branche par défaut est "master". Créer une branche crée un nouveau pointeur sans copier tout le répertoire de travail.
